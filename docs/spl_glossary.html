1 A and B registers

implemented based on "Updating the Forth Virtual Machine" by Stephen Pelc, MicroProcessor Engineering http://www.complang.tuwien.ac.at/anton/euroforth/ef08/papers/pelc.pdf

The A and B registers provide the source and destination address pointers used for block and string operations as well as providing scratch storage.

1.1 >A \ x --

Writes to the A register.

1.2 >B \ x --

Writes to the B register.

1.3 A> \ – x

Reads the A register.

1.4 B> \ – x

Reads the B register.

1.5 A@ \ – x

Read the memory pointed to by the A register.

1.6 A! \ x --

Write the memory pointed to by the A register

1.7 B@ \ – x

Read the memory pointed to by the B register.

1.8 B! \ x --

Write the memory pointed to by the B register

1.9 A@+ \ – x

Read memory pointed to by A, increment A by one cell. A post-incremented read.

1.10 B@+ \ – x

Read memory pointed to by B, increment B by one cell. A post-incremented read.

1.11 A@- \ – x

Read memory pointed to by A, decrement A by one cell. A post-decremented read.

1.12 B@- \ – x

Read memory pointed to by B, decrement B by one cell. A post-decremented read.

1.13 A!+ \ x --

Write to the memory pointed to by A, and update A.

1.14 B!+ \ x --

Write to the memory pointed to by B, and update B.

2 X and Y registers.

The X and Y registers provide indexed addressing. In Forth they can be used to implement the USER area and local frame pointers.

2.1 >X \ x --

Writes to the X register.

2.2 >Y \ x --

Writes to the Y register.

2.3 X> \ – x

Reads the X register.

2.4 Y> \ – x

Reads the Y register.

2.5 nX@ \ n – x (currently not implemented in SPL)

Read the memory pointed to by the X register plus n (literal) address units.

2.6 nX! \ x – (currently not implemented in SPL)

Write the memory pointed to by the X register plus n (literal) address units.

2.7 nY@ \ – x (currently not implemented in SPL)

Read the memory pointed to by the Y register plus n (literal) address units.

2.8 nY! \ x – (currently not implemented in SPL)

Write the memory pointed to by the Y register plus n (literal) address units

3 6502 subroutine calls

3.1 areg \ 8b --

store the low 8bit from top of stack into ea (6502 A-Reg for "call" subroutine call)

3.2 call \ addr --

jump to subroutine at addr

4 bit manipulation

4.1 and \ a b – n

bitwise AND of the two topmost stack items

4.2 or \ a b – n

bitwise OR of the two topmost stack items

4.3 xor \ a b – n

bitwise XOR (exclusive OR) of the two topmost stack items

4.4 bclr \ n b – n'

clear bit b of n

4.5 bset \ n b – n'

set bit b of n

4.6 btest \ n b – 0|1

test if bit b of n is set (ie, return the bit value)

4.7 ~ (comp) \ n - n'

comp - flip the bits of the TOS

4.8 ctoggle \ n addr --

toggle bits in byte at given address

5 Boolean logic

6 Arithmetics

6.1 abs \ n – n'

return the absolute value of the top of stack (TOS)

6.2 dabs \ d – d'

return the absolute value of a 32bit value on TOS

6.3 + (add) \ a b – a+b

add the top of stack with next of stack

6.4 d+ (dadd) \ d1 d2 – d1+d2

add two 32bit values on top of stack

6.5 +! (addstore) \ addr b --

add b to the value pointed to be addr

6.6 ++ (incaddr) \ addr --

increment 16bit word at addr by 1

6.7 c++ (inccaddr) \ addr --

increment 8bit value at addr by 1

6.8 d/ \ d0 d1 – d0/d1

double precision division

7 Strings

7.1 0trim \ addr len --

trim leading zeroes from string at addr and make it an counted string in PAD

7.2 accept \ addr – len (Apple II specific)

get a line of text in the input buffer

7.3 &. (ampdot) \ addr d --

convert 32bit number d into ASCII at addr addr must be at least 10 bytes

7.4 tooutbuf \ n --

converts text representation of n into outbuf

7.5 count \ c-addr1 – caddr2 u

return the character string specification for the counted string stored in c-addr1. c-addr2 is the address of the first character after c-addr1. u is the content of the character at caddr1, which is the length in character of the string in c-addr2.

8 Memory

8.1 @ (fetch) \ addr – n

fetch 16bit value at addr onto top of stack

8.2 d@ (double-fetch) \ addr – d

fetch a 32bit value at addr onto top of stack

8.3 c@ (char-fetch) \ addr – c

fetch a 8bit value at addr onto the top of stack

8.4 cmove \ a b n --

move n bytes from a to b

8.5 cmove> \ a b n --

move a range of bytes backwards, n bytes from a to b

9 Input/Output

9.1 at \ y x --

position cursor at location x and y

9.2 ch \ n – (Atari specific)

set horizontal cursor position

9.3 cv \ n – (Atari specific)

set vertical cursor position

9.4 cls \ – (Atari specific)

clear screen and set cursor to home position

9.5 cr \ – (Atari specific)

move to the next text line (send carriage return $9B)

10 Environment

10.1 bye \ – (Atari specific)

jump through DOS vector at $A to enter DOS

10.2 crson \ – (Atari specific)

switch cursor on

10.3 crsoff ­- (Atari specific)

switch cursor off

10.4 date \ – dd mm yy (not implemented)

return the current date from DOS
